Definición

¿Qué es la Programación Orientada a Objetos?

La POO es un paradigma de programación que organiza el software en objetos, que son instancias de clases. Cada objeto combina datos (atributos) y comportamiento (métodos) en una sola unidad, modelando el mundo real de forma intuitiva y modular.

Concepto clave

Clase vs. Objeto

Una clase es el molde o plantilla (ej: Automóvil). Un objeto es una instancia concreta de esa clase (ej: tu Toyota Corolla rojo, año 2022). La clase define qué puede hacer; el objeto es quien realmente lo hace.

Abstracción

Simplificar para enfocarse en lo que importa

Definición

La abstracción consiste en identificar las características esenciales de un objeto ignorando los detalles irrelevantes. Define qué hace un objeto, sin especificar cómo lo hace internamente. Se implementa mediante clases abstractas e interfaces.

Ejemplo en Python

from abc import ABC, abstractmethod

class Figura(ABC): # Clase abstracta
    @abstractmethod
    def area(self): pass

    @abstractmethod
    def perimetro(self): pass

class Circulo(Figura):
    def __init__(self, radio):
        self.radio = radio
    def area(self):
        return 3.14 * self.radio ** 2

Encapsulamiento

Datos seguros, acceso controlado

Definición

El encapsulamiento agrupa datos y métodos relacionados dentro de una clase y restringe el acceso directo al estado interno del objeto. Los datos privados solo son accesibles mediante métodos públicos (getters/setters), protegiendo la integridad del objeto.

Ejemplo en Java

public class CuentaBancaria {
    private double saldo; // atributo privado

    public double getSaldo() {
        return saldo;
    }

    public void depositar(double monto) {
        if (monto > 0) saldo += monto;
    }
}

Herencia

Reutilizar y extender comportamiento

Definición

La herencia permite crear una nueva clase (subclase) que hereda atributos y métodos de una clase existente (superclase). Promueve la reutilización de código y establece relaciones jerárquicas del tipo "es un" (ej: un Perro es un Animal).

Ejemplo en Python

class Animal:
    def __init__(self, nombre):
        self.nombre = nombre
    def respirar(self):
        return "Inhala... exhala..."

class Perro(Animal): # hereda de Animal
    def ladrar(self):
        return f"{self.nombre} dice: ¡Guau!"

rex = Perro("Rex")
rex.respirar() # hereda el método

Polimorfismo

Muchas formas, una sola interfaz

Definición

El polimorfismo permite que objetos de distintas clases respondan al mismo mensaje de maneras diferentes. Puede ser por sobrecarga (mismo nombre, distintos parámetros) o por sobreescritura (redefinir el método en la subclase). Permite escribir código más genérico y flexible.

Ejemplo en Python

class Gato(Animal):
    def hablar(self):
        return "¡Miau!"

class Perro(Animal):
    def hablar(self):
        return "¡Guau!"

animales = [Gato(), Perro(), Gato()]
for a in animales:
    print(a.hablar()) # polimorfismo

¿Por qué los lenguajes dan soporte a la POO?

Los lenguajes modernos adoptaron la POO porque resuelve problemas reales de desarrollo de software a gran escala. Aquí las razones principales:

  • 01

    Modelado del mundo real

    Los objetos mapean directamente a entidades del mundo real (Usuario, Factura, Producto), haciendo el código más intuitivo y comprensible para equipos multidisciplinarios.

  • 02

    Reutilización de código

    La herencia y la composición permiten reutilizar clases existentes en lugar de reescribir lógica. Esto reduce errores y acelera el desarrollo.

  • 03

    Mantenibilidad

    El encapsulamiento aísla los cambios internos de una clase sin afectar al resto del sistema, facilitando la corrección de bugs y la evolución del software.

  • 04

    Escalabilidad

    Los sistemas POO se pueden ampliar añadiendo nuevas clases sin modificar las existentes (Principio Abierto/Cerrado), ideal para proyectos que crecen en el tiempo.

  • 05

    Trabajo en equipo

    Las interfaces y abstracciones permiten que distintos equipos trabajen en paralelo: uno implementa la clase, otro la consume, sin dependencias de implementación interna.

  • 06

    Patrones de diseño

    Los patrones de diseño clásicos (Singleton, Factory, Observer, Strategy...) se basan directamente en los pilares POO, ofreciendo soluciones probadas a problemas comunes.
